# ERC20ClaimPhases

Allow other wallets to claim/mint tokens from the contract under the criteria of claim conditions by
implementing `ERC1155`, `LazyMint` and `Drop`. 

Unlike [`ERC1155ClaimCondition`](/solidity/extensions/erc1155claimconditions), this extension allows you to set *multiple* claim phases.

This is an extension which is detectable in the dashboard if the smart contract implements the 
[ERC1155](/solidity/extensions/erc1155), [LazyMint](/solidity/extensions/lazymint) and [Drop](/solidity/extensions/drop) extensions.

## Usage

This is an example smart contract demonstrating how to use this extension.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "@thirdweb-dev/contracts/eip/ERC1155.sol";
import "@thirdweb-dev/contracts/extension/LazyMint.sol";
import "@thirdweb-dev/contracts/extension/DropSinglePhase.sol";

contract Contract is ERC1155, LazyMint, DropSinglePhase {
    constructor(
        string memory _name,
        string memory _symbol
    )
        ERC1155(
            _name,
            _symbol
        )
    {}

    function _canLazyMint() internal view override returns (bool) {
        // Your custom implementation here
    }

    function _canSetClaimConditions() internal view override returns (bool) {
        // Your custom implementation here
    }

    function _collectPriceOnClaim(
        address _primarySaleRecipient,
        uint256 _quantityToClaim,
        address _currency,
        uint256 _pricePerToken
    ) internal virtual override {
        // Your custom implementation here
    }

    function _transferTokensOnClaim(address _to, uint256 _quantityBeingClaimed)
        internal
        virtual
        override
        returns (uint256 startTokenId)
    {
        // Your custom implementation here
    }
}
```
